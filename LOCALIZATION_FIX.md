# ✅ Исправление локализации названий блюд

## Проблема
Названия блюд сохранялись в админ-панели на трех языках, но не отображались на клиентской стороне при переключении языка.

## Причина
В `BarPage.jsx` и `PizzaPage.jsx` при трансформации данных с сервера не передавались поля `name_ru`, `name_en`, `name_kk`.

## Что исправлено
✅ Добавлены поля `name_ru`, `name_en`, `name_kk` в трансформацию данных в `BarPage.jsx`  
✅ Добавлены поля `name_ru`, `name_en`, `name_kk` в трансформацию данных в `PizzaPage.jsx`  
✅ `MenuPage.jsx` использует данные напрямую, там все работает корректно  

## Что нужно сделать

### 1. Пересоберите frontend (если используете production build)
```bash
cd frontend
npm run build
```

### 2. Обновите страницу
Нажмите `Ctrl + Shift + R` для жесткой перезагрузки страницы

### 3. Проверьте работу
1. Откройте страницу Бар или Пицца
2. Переключите язык (RU → EN → KK)
3. Названия блюд должны меняться в соответствии с выбранным языком

## Как это работает

Когда вы переключаете язык, компонент `DishCard` автоматически выбирает правильное название:

```javascript
const getDishName = (dish) => {
  const lang = i18n.language;
  if (lang === 'en' && dish.name_en) return dish.name_en;
  if (lang === 'kk' && dish.name_kk) return dish.name_kk;
  return dish.name_ru || dish.name; // Fallback на русский
};
```

Если перевод не указан, отображается название на русском языке.
