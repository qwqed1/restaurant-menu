# Обновление системы локализации - Названия блюд

## Что было сделано

### 1. База данных
- ✅ Добавлены поля `name_ru`, `name_en`, `name_kk` в таблицу `dishes`
- ✅ Существующие названия скопированы во все языковые поля
- ✅ Миграция выполнена успешно

### 2. Backend API
- ✅ Обновлены эндпоинты для получения блюд (включают все языковые поля)
- ✅ Обновлен эндпоинт создания блюда (требует `name_ru`, опционально `name_en` и `name_kk`)
- ✅ Обновлен эндпоинт редактирования блюда (поддерживает все языковые поля)

### 3. Админ-панель
- ✅ Добавлены три поля для ввода названия блюда:
  - **Название на русском** (обязательное)
  - **Название на английском** (опциональное)
  - **Название на казахском** (опциональное)
- ✅ Форма автоматически заполняет поле `name` значением из `name_ru`

### 4. Frontend (клиентская часть)
- ✅ Обновлен компонент `DishCard` - отображает название на выбранном языке
- ✅ Обновлен компонент `DishModal` - отображает название на выбранном языке
- ✅ Автоматический fallback на русский язык, если перевод отсутствует

## Как использовать

### В админ-панели:
1. Откройте раздел "Управление блюдами"
2. При создании/редактировании блюда заполните:
   - **Название на русском** - обязательно
   - **Название на английском** - по желанию
   - **Название на казахском** - по желанию
3. Сохраните изменения

### На клиентской стороне:
- Названия блюд автоматически меняются при переключении языка
- Если перевод не указан, отображается название на русском языке

## Структура полей

### Таблица dishes:
```sql
name          TEXT         -- Основное название (копия name_ru)
name_ru       TEXT NOT NULL -- Название на русском (обязательно)
name_en       TEXT         -- Название на английском (опционально)
name_kk       TEXT         -- Название на казахском (опционально)
```

## Примечания
- Все существующие блюда автоматически получили значения во всех языковых полях
- Система полностью обратно совместима
- Локализация работает для всех страниц: Кухня, Бар, Пицца
